# Хеш-функції в Python hashlib

## MD5 (Message Digest Algorithm 5)

**Характеристики:**

- Довжина: 128 біт (32 hex символи)
- Швидкість: дуже швидкий
- Рік: 1991

**Чому небезпечний:**

- Знайдені колізії (два різні входи → один хеш)
- Існують rainbow tables для швидкого зламу паролів
- Вразливий до атак

**Використання:**

- ❌ НЕ для паролів
- ❌ НЕ для криптографії
- ✅ Контрольні суми файлів (якщо безпека не критична)
- ✅ Швидке порівняння даних

---

## SHA-1 (Secure Hash Algorithm 1)

**Характеристики:**

- Довжина: 160 біт (40 hex символів)
- Швидкість: швидкий
- Рік: 1995

**Чому небезпечний:**

- Офіційно зламаний у 2017 році (Google знайшов колізію)
- Deprecated у більшості систем

**Використання:**

- ❌ НЕ для нових проектів
- ⚠️ Legacy системи (Git ще використовує, але переходить на SHA-256)

---

## SHA-256 (SHA-2 family)

**Характеристики:**

- Довжина: 256 біт (64 hex символи)
- Швидкість: швидкий
- Рік: 2001
- Алгоритм: Merkle–Damgård

**Чому безпечний:**

- Немає відомих практичних атак
- Широко використовується та перевірений часом
- Стандарт в індустрії

**Використання:**

- ✅ Bitcoin, Blockchain
- ✅ SSL/TLS сертифікати
- ✅ Цифрові підписи
- ✅ Контрольні суми
- ⚠️ Паролі (краще bcrypt/argon2, але SHA-256 + salt теж ок)

---

## SHA-512 (SHA-2 family)

**Характеристики:**

- Довжина: 512 біт (128 hex символів)
- Швидкість: швидкий на 64-бітних системах
- Рік: 2001
- Алгоритм: Merkle–Damgård (як SHA-256)

**Відмінності від SHA-256:**

- Довший хеш → більша стійкість до колізій
- Ефективніший на 64-бітних процесорах
- Більше обчислень → трохи повільніший на 32-біт

**Використання:**

- ✅ Все те саме, що SHA-256
- ✅ Коли потрібна максимальна безпека
- ✅ Системи з 64-бітною архітектурою

---

## BLAKE2b

**Характеристики:**

- Довжина: до 512 біт (128 hex символів за замовчуванням)
- Швидкість: швидший за SHA-256
- Рік: 2012
- Алгоритм: базується на ChaCha

**Переваги:**

- Швидший за MD5, але безпечний як SHA-3
- Підтримка ключів (HMAC вбудовано)
- Паралелізація

**Використання:**

- ✅ Сучасна альтернатива SHA-256
- ✅ Коли потрібна швидкість + безпека
- ✅ Argon2 (алгоритм для паролів) використовує BLAKE2
- ⚠️ Менш поширений, ніж SHA-2

---

## SHA3-256 (Keccak)

**Характеристики:**

- Довжина: 256 біт (64 hex символи)
- Швидкість: трохи повільніший за SHA-256 на CPU
- Рік: 2015
- Алгоритм: Sponge function (Keccak)

**Чим відрізняється від SHA-256:**

- **Зовсім інша математика**: SHA-256 використовує Merkle–Damgård, SHA3 — sponge function
- **Резервний варіант**: якщо знайдуть вразливість у SHA-2, SHA3 залишиться безпечним
- **Різна внутрішня структура**: стійкий до атак довжини повідомлення

**Використання:**

- ✅ Новий стандарт NIST (2015)
- ✅ Ethereum використовує Keccak-256
- ✅ Коли потрібна альтернатива SHA-2
- ⚠️ Ще не так широко підтримується, як SHA-2

## Рекомендації

### Для паролів:

1. **bcrypt** або **argon2** (спеціалізовані бібліотеки)
2. Якщо немає доступу: SHA-256 + salt + кілька ітерацій

### Для загального хешування:

1. **SHA-256** — найбезпечніший вибір (стандарт)
2. **BLAKE2b** — якщо важлива швидкість
3. **SHA3-256** — якщо потрібна альтернатива SHA-2

### Чого уникати:

- ❌ MD5 для будь-якої безпеки
- ❌ SHA-1 для нових проектів
- ❌ Хешування паролів без salt
- ❌ Одна ітерація для паролів

## Приклади використання

```python
import hashlib

# Для файлів (контрольна сума)
data = open('file.txt', 'rb').read()
checksum = hashlib.sha256(data).hexdigest()

# Для унікальних ID
user_id = hashlib.sha256(f"{email}{timestamp}".encode()).hexdigest()

# Для паролів (краще bcrypt, але якщо немає)
import os
salt = os.urandom(32)
password_hash = hashlib.pbkdf2_hmac('sha256', password.encode(), salt, 100000)
```
