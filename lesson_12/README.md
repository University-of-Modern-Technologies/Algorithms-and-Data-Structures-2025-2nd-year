# Lesson 12 – NetworkX Basics

Набір прикладів у цій директорії показує шлях від найпростіших операцій з графами до невеликого case study. Усі скрипти розраховані на Python 3.12 та бібліотеки `networkx`, `matplotlib`.

## Як запускати

Кожен файл самодостатній, виводить потрібні пояснення в консоль і малює граф. Якщо потрібних пакетів нема, встанови їх командою `pip install networkx matplotlib`.

## Що розглядається в прикладах

| Файл | Основна тема | Ключові прийоми |
| --- | --- | --- |
| `example01.py` | Створення та редагування графа | Додаємо/видаляємо вузли й ребра, дивимось статистику, будуємо перший неорієнтований граф. |
| `example02.py` | Орієнтовані графи й атрибути | Використання `DiGraph`, атрибути вузлів/ребер, доступ до них, малювання зі стрілками та підписами доріг. |
| `example03.py` | Візуалізація через атрибути вузлів | Атрибут `type` визначає кольори, показуємо, як програмно генерувати палітру та підписувати ваги ребер. |
| `example04.py` | Аналітика на зваженому графі | Додаємо ваги, обчислюємо degree/betweenness/closeness, будуємо найкоротші шляхи (звичайні та з вагами), малюємо граф з підписами. |
| `example04a.py` | Компоненти зв’язності та критичні точки | На графі A–E демонструємо `nx.connected_components`, `nx.bridges`, `nx.articulation_points` і що станеться після видалення мосту C–D. |
| `example05.py` | Circular layout | Показуємо розкладку по колу для повного графа. |
| `example06.py` | Random layout | Генеруємо випадкове розташування вузлів і порівнюємо з іншими layout. |
| `example07.py` | Shell layout | Налаштовуємо концентричні “камери”, щоб показати різний рівень важливості вузлів. |
| `example08.py` | Spring layout | Пояснюємо “фізичну” розкладку, яка використовується за замовчуванням. |
| `example09.py` | DFS/BFS деревами NetworkX | Через `nx.dfs_tree` та `nx.bfs_tree` порівнюємо стратегії обходу й показуємо три діаграми. |
| `example10.py` | Власні реалізації DFS/BFS | Ручні функції обходу поруч із NetworkX, звіряємо порядки відвідування, малюємо граф. |
| `example11.py` | Case study соцмережі | Структуровані дані про персонажів, degree/betweenness на зважених зв’язках, аналіз пріоритетів, динамічна візуалізація з кольорами, товщиною ребер та підписами. |
