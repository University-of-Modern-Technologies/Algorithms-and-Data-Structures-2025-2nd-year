# Тема 24. Алгоритмічна складність, наближені алгоритми

Цей проєкт містить практичні приклади до теми "Алгоритмічна складність, наближені алгоритми" для курсу алгоритмів.

## Файли проєкту

### 01_graph_coloring.py
**Розфарбування графа (Graph Coloring)**

Демонструє перебірний алгоритм розфарбування графа. Алгоритм перевіряє всі можливі комбінації кольорів для вершин графа та знаходить валідне розфарбування, де суміжні вершини мають різні кольори.

**Складність:** O(k^n), де k - кількість кольорів, n - кількість вершин.

**Функції:**
- `is_valid_coloring()` - перевіряє, чи є розфарбування валідним
- `color_graph()` - знаходить розфарбування графа методом повного перебору

---

### 02_greedy_set_cover.py
**Жадібний алгоритм покриття множини (Greedy Set Cover)**

Реалізує жадібний наближений алгоритм для задачі покриття множини. На кожному кроці вибирає підмножину, яка покриває найбільшу кількість ще непокритих елементів.

**Складність:** O(n * m), де n - кількість елементів універсальної множини, m - кількість підмножин.

**Апроксимація:** O(log n) - алгоритм гарантує розв'язок, який не більше ніж у log(n) разів гірший за оптимальний.

**Функції:**
- `greedy_set_cover()` - знаходить наближене покриття множини жадібним алгоритмом

---

### 03_knapsack_fptas.py
**FPTAS для задачі про рюкзак (Fully Polynomial Time Approximation Scheme)**

Реалізує FPTAS (Fully Polynomial Time Approximation Scheme) для задачі про рюкзак. Алгоритм масштабує вартості предметів та використовує динамічне програмування для отримання наближеного розв'язку з заданою точністю.

**Складність:** O(n² / ε), де n - кількість предметів, ε - параметр точності.

**Апроксимація:** (1 - ε) - алгоритм гарантує розв'язок, який не менше ніж (1 - ε) від оптимального.

**Функції:**
- `knapsack_fptas()` - знаходить наближений розв'язок задачі про рюкзак з заданою точністю

---

### 04_subset_sum_bruteforce.py
**Перебірний алгоритм для задачі про підсумок підмножини (Subset Sum Brute Force)**

Демонструє точний розв'язок задачі про підсумок підмножини методом повного перебору. Генерує всі можливі підмножини та знаходить ту, яка має максимальну суму, що не перевищує задану межу.

**Складність:** O(2^n), де n - кількість елементів.

**Функції:**
- `subset_sum_bruteforce()` - знаходить точний розв'язок задачі про підсумок підмножини методом перебору

---

### 05_subset_sum_fptas.py
**FPTAS для задачі про підсумок підмножини (Subset Sum FPTAS)**

Реалізує FPTAS для задачі про підсумок підмножини. Алгоритм масштабує значення елементів та використовує динамічне програмування для отримання наближеного розв'язку з заданою точністю.

**Складність:** O(n² / ε), де n - кількість елементів, ε - параметр точності.

**Апроксимація:** (1 - ε) - алгоритм гарантує розв'язок, який не менше ніж (1 - ε) від оптимального.

**Функції:**
- `subset_sum_fptas()` - знаходить наближений розв'язок задачі про підсумок підмножини з заданою точністю

---

## Основні концепції

### Перебірні алгоритми
- **01_graph_coloring.py** - демонструє повний перебір для задачі розфарбування графа
- **04_subset_sum_bruteforce.py** - демонструє повний перебір для задачі про підсумок підмножини

### Жадібні наближені алгоритми
- **02_greedy_set_cover.py** - жадібний алгоритм з гарантованим коефіцієнтом апроксимації

### FPTAS (Fully Polynomial Time Approximation Scheme)
- **03_knapsack_fptas.py** - FPTAS для задачі про рюкзак
- **05_subset_sum_fptas.py** - FPTAS для задачі про підсумок підмножини

FPTAS алгоритми дозволяють отримати наближений розв'язок з будь-якою заданою точністю за поліноміальний час, що робить їх практично корисними для NP-складних задач.
